# AnnotaMD v2.10.1

本版本重点提升启动速度、大文档编辑、批注、表格和目录浏览性能，并将 Electron 升级到 43.1.1。

## 改进

### 启动与基础性能

- Electron 从 42.1.0 升级到 43.1.1，获得新版 Chromium、Node.js 和 V8 的性能与兼容性改进。
- 减少首次启动需要加载的资源：初始 JavaScript 体积约减少 78%，gzip 后约减少 75%，初始 CSS 约减少 52%。
- 同一台设备连续启动 10 次的对比测试中，启动耗时中位数由约 1,001 ms 降至 933 ms，较慢样本由约 1,336 ms 降至 966 ms。实际效果会随设备和文档规模变化。

### 大文档与输入响应

- 减少输入时同步执行的字数统计、目录刷新和撤销记录处理，长文档连续输入更加流畅。
- 1 MB 文档修改状态检查的对比耗时由约 16.1 ms 降至 1.61 ms，约减少 90%。
- 大量表格文档的打开速度明显提升。在包含 51 张表格、1,513 个单元格的测试文档中，打开阶段由约 4,063 ms 降至 430 ms，约提升 89%。
- 避免新文档首次打开时重复构建内容，降低首次加载等待时间。

### 批注与表格

- 减少批注锚点、文本高亮和评论栏定位中的重复计算与重复保存，大量批注下的滚动和编辑响应更稳定。
- 合并同一帧内的批注卡片布局更新，降低评论栏频繁跳动和卡顿的概率。
- 减少表格吸顶标题的重复测量，长表格滚动更顺畅。

### 目录浏览

- 优化大型目录的读取、排序和增量更新。1,000 个 Markdown 文件的处理耗时由约 11.22 ms 降至 2.82 ms，10,000 个文件由约 309.17 ms 降至 19.59 ms。
- macOS 本地目录默认使用原生文件变化监听，文件新增、删除和重命名的响应更及时；网络盘和云盘仍保留兼容模式。

### Agent 设置

- 新安装默认开启“允许 Agent 访问文档评论”，配置完成的本地 Agent 可以直接读取和处理保存在本机的评论。
- 已升级用户原有的开关选择不会被覆盖，仍可随时在“设置 → Agent”中关闭。

## 修复

- 修复表格密集文档打开时间异常增长的问题。
- 修复连续输入时重复生成 Markdown 和重复复制编辑状态造成的卡顿。
- 修复批注位置未变化时仍被反复保存的问题。
- 修复大量目录事件集中发生时可能造成的界面阻塞。
- 恢复 Linux 版本下载，提供 AppImage、deb、rpm、snap 和 tar.gz 安装包。

## macOS 安装提示

当前发行构建使用免费 ad-hoc 签名，暂未使用 Apple Developer ID 签名和公证。将 AnnotaMD 拖入“应用程序”后，如 macOS 阻止启动，可执行：

```bash
xattr -cr /Applications/AnnotaMD.app
```

也可以在“系统设置 → 隐私与安全性”中选择“仍要打开”。

## 下载校验

所有安装包均记录在 Release 的 `SHA256SUMS.txt` 中，可在下载目录执行：

```bash
sha256sum -c SHA256SUMS.txt --ignore-missing
```
