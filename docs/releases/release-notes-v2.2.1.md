# v2.2.1

本版本继续完善飞书风格的 Markdown 快编辑体验，并集中修复文字块转换、列表菜单定位和表格操作问题。

## 重点更新

- **统一文字块转换**：正文、H1–H6、引用、有序列表、无序列表、待办和高亮块共用同一套转换入口，并支持双向转换。
- **列表项级编辑**：补充列表拆分、相邻列表合并和多块转换基础能力，减少整组列表与单项转换时的内容错位。
- **高亮容器块**：新增可折叠的高亮块及 Markdown 往返转换支持，内部标题不进入外部目录。
- **块菜单修复**：有序列表、无序列表和待办项的快编辑按钮为列表标记预留空间，不再遮挡序号、圆点或复选框。
- **表格工具栏优化**：统一批注、删除等操作的图标顺序和配置，并补充定向回归测试。
- **可复用验证流程**：新增快速验证脚本和项目开发经验说明，后续可按功能域快速运行菜单、表格、批注及类型检查。

## macOS 安装提示

当前构建使用免费 ad-hoc 签名，暂未使用 Apple Developer ID 签名和公证。将 AnnotaMD 拖入“应用程序”并尝试打开后，如 macOS 阻止启动：

1. 打开“系统设置 → 隐私与安全性”；
2. 向下滚动到“安全性”；
3. 找到被阻止的 AnnotaMD，点击“仍要打开”；
4. 在确认窗口中再次点击“打开”。

完成一次后，后续可以像普通应用一样双击启动。
