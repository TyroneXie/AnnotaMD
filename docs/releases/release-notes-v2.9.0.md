# AnnotaMD v2.9.0

本版本重点完善本地 Markdown 的批注与 Agent 协作：批注完整保存在本机 SQLite 中，正文仍是干净、标准的 Markdown；配置后的 Agent 可以理解评论、回复问题，也可以按精确锚点直接修改文档。

## 本地私有批注

- 选区批注、全文批注、回复和解决状态统一保存在应用本机 SQLite 数据库中，不写入 Markdown，也不在文档旁创建额外文件夹。
- 文件复制或另存为后得到干净正文，不会意外继承原文件批注。
- 同一文件重命名或移动后仍保留批注；文件删除后批注立即隐藏，并在七天后自动清理。
- 批注通过编辑器事务中的精确位置跟随正文，不依赖搜索相同文字。文档中即使存在大量重复内容，也能区分批注实际指向的位置。
- 修改批注前后或移动章节时批注继续跟随；被批注的文字本身发生变化时，原批注自动失效，避免错误关联。

## Agent 与 MCP 协作

- 新增 AnnotaMD MCP 服务，可发现存在待处理评论的文档，并同时读取完整 Markdown、选区批注、全文批注和线程回复。
- Agent 可以根据评论判断是修改正文、直接回复问题，还是先向用户确认；精确修改成功后会保留处理记录并标记为已解决。
- 所有写操作使用文档 revision 防止覆盖较新的内容，并通过评论 ID 和真实锚点处理重复文字。
- 设置中提供 ChatGPT 与 Claude Code 的检测和接入，也可复制标准 STDIO MCP 配置用于其他本地 Agent。
- 关闭“允许 Agent 访问文档评论”后，SQLite 私有批注立即停止对外服务；Agent 原有的本地文件读取权限不受影响。

## 界面与侧栏改进

- 右侧批注栏默认更窄，标题栏上下间距更紧凑，并统一显示“批注（数量）”、MCP 入口和收起按钮。
- Agent 回复在线程中带有清晰标识，已解决评论保留处理记录，方便用户确认修改结果。
- 已打开文件列表使用侧栏剩余高度；“打开文件夹”按钮始终位于剩余空白区域的视觉中心。
- 文档目录支持原生横向滚动，长标题可通过触控板或底部滚动条完整查看。
- 修复设置窗口隐藏或最小化后无法正常重新打开，以及 Agent 列表延迟出现、检测文案闪烁等问题。

## macOS 安装提示

当前构建使用免费 ad-hoc 签名，暂未使用 Apple Developer ID 签名和公证。将 AnnotaMD 拖入“应用程序”后，如 macOS 阻止启动，可执行：

```bash
xattr -cr /Applications/AnnotaMD.app
```

也可以在“系统设置 → 隐私与安全性”中选择“仍要打开”。
