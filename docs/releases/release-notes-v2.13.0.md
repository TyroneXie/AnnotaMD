# AnnotaMD v2.13.0

本版本扩展了 AnnotaMD 的本地文档审阅工作流：现在可以跨段落创建一条完整批注，直接查看 Agent 的实时连接状态，并在更稳定的评论栏中完成多轮讨论、导航和逐条编辑。批注仍只保存在本机，不会写入 Markdown 正文；评论是否解决继续由用户决定。

## 新增

### 跨块批注与编辑

- 鼠标选区跨越多个段落或列表项时，可以创建一条批注；高亮和锚点覆盖完整选区，评论栏只生成一个线程。
- 多块选区可以一次应用兼容的文字格式，也可以转换为段落、标题、列表、引用或代码块。
- 跨块格式化会跳过不兼容的代码块，避免批量操作破坏 Markdown 结构。

### Agent 实时连接

- “设置 → Agent”会在 Agent 完成 MCP 握手后显示实时连接状态、连接时长和最近连接记录，不再依赖手动刷新。
- Codex 与 Claude Code 可以安装配套评论工作流；其他支持本地 STDIO MCP 的 Agent 可通过完整配置接入。
- Agent 可以列出评论、读取完整线程并在线程中回复；最后一条消息来自 Local 的线程会被视为仍需 Agent 处理。
- Agent 不会自动解决或删除评论，最终状态始终由用户确认。

### 评论导航与多轮讨论

- 评论栏新增上一条、下一条导航，可快速在正文锚点之间移动。
- 未选中的长线程默认只显示最新消息，选中后可展开全部内容并随时收起。
- 每条 Local 消息都可以独立编辑或删除；Agent 消息保持只读，来源和时间更加清晰。

## 改进

- 选中的评论卡片优先与对应正文锚点对齐，周围评论向上下两侧避让，密集批注时仍能保持对应关系。
- 超出可见空间的长线程在卡片内部使用原生滚动和惯性，不会带动正文跳转；切换到其他线程后再次打开会从顶部开始。
- 正文与评论栏共用右侧唯一滚动条，滚动停止后自动隐藏轨道和滑块，阅读区域更加整洁。
- 被批注文字发生局部插入、删除或替换时继续保留评论；只有完整选区被删除或整体替换时才移除锚点。
- 新建评论、展开线程、回复输入和评论导航不再抢走正文当前阅读位置。

## 修复

- 修复长线程首次滚动时卡片高度变化、评论栏停顿或正文被意外带动的问题。
- 修复在线程底部点击回复输入框时，正文跳回上方锚点的问题。
- 修复切换评论后复用上一条线程阅读位置，以及展开、收起时正文滚动的问题。
- 修复在已有批注范围内再次创建评论时，新输入框被旧评论挤出视口的问题。
- 修复正文锚点离开视口后评论卡片仍悬停，以及行内代码两侧批注下划线断开的问题。
- 修复普通单块选区应用粗体等格式后，格式工具栏意外关闭的问题。
- 修复清空代码块标题后语言候选浮层跳到窗口左上角的问题。

## macOS 安装提示

当前发行构建使用免费 ad-hoc 签名，暂未使用 Apple Developer ID 签名和公证。将 AnnotaMD 拖入“应用程序”后，如 macOS 阻止启动，可执行：

```bash
xattr -cr /Applications/AnnotaMD.app
```

也可以在“系统设置 → 隐私与安全性”中选择“仍要打开”。

## 下载校验

所有安装包均记录在 Release 的 `SHA256SUMS.txt` 中，可在下载目录执行：

```bash
sha256sum -c SHA256SUMS.txt --ignore-missing
```
