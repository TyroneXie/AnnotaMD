# AnnotaMD v2.14.0

本版本让批注与本地 Agent 的协作更直接：配置 Agent CLI 后，可以从新建批注或线程回复中直接发送任务，并在同一线程查看执行过程与结果。批注栏同时明确区分“可以直接发送”和“已有 Agent 应用通过 MCP 实时连接”两种状态，避免把配置记录误认为当前可访问。

## 新增

### 从批注栏直接调用 Agent

- 在“设置 → Agent”中配置本地 Agent CLI 后，新建批注和线程回复都可以直接发送给选定的 Agent。
- 执行状态和结果会回到当前评论线程，便于围绕同一处文档内容继续讨论，不必在应用之间反复复制。
- 批注栏顶部同时展示直接发送通道和 MCP 实时连接通道；只有 Agent 应用当前完成 MCP 连接时，第二个状态才会显示为可用。

## 改进

### 长评论阅读

- 点击评论卡片时，只有内容确实被截断且卡片高度不足批注栏可视高度的三分之二，才会自动放大。
- 已经较高的长评论继续在卡片内部滚动，不会反复改变正文位置。
- 多轮评论的“收起”操作固定在卡片底部，阅读长内容时无需滚动到线程末尾寻找。

### 状态与界面

- Agent 状态浮层使用两行状态和独立圆点，清楚说明批注栏直接发送与 MCP 应用访问能力。
- 点击浮层外部即可关闭；窄批注栏中浮层不会被左侧边界遮挡，标题栏控件也保持对齐。
- 引用块在不同编辑器主题中的外观更加一致。

## 修复

- 修复已保存 MCP 配置或近期连接记录被误判为 Agent 应用当前在线的问题。
- 修复手动检查更新后没有结果反馈的问题；当前 macOS ad-hoc 签名版本会明确引导手动安装，不再尝试无法完成的自动安装。
- 修复 Agent 状态浮层持续停留、窄栏内被裁切，以及批注标题栏图标位置不齐的问题。

## macOS 安装提示

当前发行构建使用免费 ad-hoc 签名，暂未使用 Apple Developer ID 签名和公证。将 AnnotaMD 拖入“应用程序”后，如 macOS 阻止启动，可执行：

```bash
xattr -cr /Applications/AnnotaMD.app
```

也可以在“系统设置 → 隐私与安全性”中选择“仍要打开”。

## 下载校验

所有安装包均记录在 Release 的 `SHA256SUMS.txt` 中，可在下载目录执行：

```bash
sha256sum -c SHA256SUMS.txt --ignore-missing
```
