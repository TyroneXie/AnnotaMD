# AnnotaMD v2.10.0

本版本聚焦桌面端性能。底层运行时升级到 Electron 43.1.1，同时针对启动、输入、批注、表格密集长文档和大型目录逐项清理应用层热点；不包含数据格式、设置或插件接口的破坏性迁移。

## Electron 43 与启动性能

- Electron 从 42.1.0 直接升级到 43.1.1，使用新版 Chromium、Node.js 和 V8，并获得 Electron 43 的主进程启动快照、preload 字节码缓存及沙箱 renderer 启动优化。
- 编辑器、设置容器和各设置分类改为按路由加载，Element Plus 只加载实际使用的组件与样式。
- renderer 初始 JavaScript 由约 7.34 MB 降至 1.62 MB，gzip 由约 1.52 MB 降至 0.379 MB，分别减少约 78% 和 75%；初始 CSS 减少约 52%。
- 编辑窗口在路由完成初始化后再显示，避免懒加载期间出现空白窗口；主进程日志改为异步写入，评论服务、MCP 桥及外部客户端探测不再阻塞关键启动路径。
- 同机 production 构建的 10 次启动样本中，p50 由约 1,001 ms 降至 933 ms，p95 由约 1,336 ms 降至 966 ms。该数据用于相同环境下的相对比较。

## 大文档与输入响应

- 每次输入直接复用编辑器已经产生的状态快照，避免重复深拷贝和重复生成 Markdown。
- 字数统计与目录刷新在输入停止后合并执行，完整撤销历史只在保存、切换标签和进入源码模式等需要的时机序列化。
- dirty 状态的 1 MB 文本哈希微基准由约 16.1 ms 降至 1.61 ms，成本降低约 90%。
- 修复表格单元格创建时反复深拷贝并遍历整篇文档的近似 O(n²) 热点，并避免新文档在首次打开时构建两遍。
- 表格密集验证文档 `design-2026-06-30-review.md` 为 137,008 字节，包含 51 张表格、1,513 个单元格和 1,894 个可编辑叶子；在已启动的 production Electron 中，打开阶段由 4,063 ms 降至 430 ms，改善约 89.4%，最终 DOM 与 Markdown 往返内容一致。

## 批注、表格与目录

- 批注锚点只在坐标实际变化时写入，连续持久化任务自动合并；批注高亮、文本校验和评论栏定位复用同一份 DOM 索引与 Range。
- 批注卡片布局按帧合并并缓存高度；sticky table header 只在几何失效时重新测量，长表格滚动不再随前方表格数量增加布局读取。
- 打开目录时只读取 Markdown 文件元数据，不再预读每个文件的完整正文；16 ms 内的目录事件合并为一次 IPC，并在 renderer 中批量插入、统一排序。
- 文件树微基准中，1,000 个文件由约 11.22 ms 降至 2.82 ms，10,000 个文件由约 309.17 ms 降至 19.59 ms。
- macOS 默认使用原生文件事件，减少空闲轮询；网络盘和云盘仍可通过现有设置显式启用 polling 回退。

## Agent 默认设置

- 新安装默认开启“允许 Agent 访问文档评论”，配置完成的本地 Agent 可直接读取和处理保存在本机的评论。
- 已有用户的持久化选择不会被覆盖，仍可随时在“设置 → Agent”中关闭该能力。

## macOS 安装提示

当前发行构建使用免费 ad-hoc 签名，暂未使用 Apple Developer ID 签名和公证。将 AnnotaMD 拖入“应用程序”后，如 macOS 阻止启动，可执行：

```bash
xattr -cr /Applications/AnnotaMD.app
```

也可以在“系统设置 → 隐私与安全性”中选择“仍要打开”。
