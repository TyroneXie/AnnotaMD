# AnnotaMD v2.11.0

本版本重点完善所见即所得编辑体验：新增层级标题编号、丰富块类型转换和智能链接标题视图，并修复长文档代码块行号、链接悬浮工具栏及多种粘贴边界问题。

## 新增

### 标题编号

- 在任意 H1–H6 标题中输入 `1.`、`1.1.` 或对应中文句号后按空格，可生成独立的蓝色层级编号。
- 后续同级标题会按上下文自动递增，子标题会继承父级编号；未编号标题不会错误中断已有序列。
- 编号可单独点击，并支持继续编号、重新开始编号和设置编号值。

### 块编辑与文字格式

- 文字、H1–H6、代码块、引用、有序/无序列表、待办和高亮块之间可直接转换。
- 选中文本后可使用加粗、斜体、删除线、下划线、颜色和评论等常用操作；代码块中的文本选择也能调出相应工具栏。
- 跨越代码块等结构块的不兼容选区会给出明确提示，避免只转换一部分内容。

### 智能链接标题视图

- 整行粘贴网页地址后会先显示原链接和加载状态，再在限定时间内自动获取网页标题与站点图标。
- 获取成功后自动显示标题；获取失败或超时则继续保留可用的原链接，不阻塞编辑。
- 独立链接可在 Link View 与 Title View 之间往返切换，已经取得的标题不会丢失。
- 给划选文字添加的链接保持普通行内链接，不显示站点图标或视图切换，避免干扰正文排版。

## 改进

- 块菜单更紧凑，图标、间距、圆角和阴影更加统一，并支持键盘打开、方向键移动和回车执行。
- H1–H6 保持直接平铺显示，不把低层级标题折叠到二级菜单。
- 分隔线按独立块选中和操作，不再把 Markdown 标记显示成不可编辑的三个点。
- 缩小待办圆圈，隐藏无实际作用的块链接入口，并让表格单元格中的 `<br>` 正常显示为换行。

## 修复

- 修复长代码块首次加载时行号越过代码块、覆盖后续表格和正文的问题。
- 修复标题编号跳号、继承错误以及部分输入格式无法规范化的问题。
- 修复鼠标从链接移动到悬浮工具栏、快速切换不同链接或在链接内编辑时工具栏偶发消失的问题。
- 修复在链接可见文字末尾按回车后 Markdown 链接被拆坏的问题。
- 修复带根路径 `/` 的网址、浏览器复制的无协议显示文本以及部分动态网页无法进入标题视图的问题。
- 修复站点图标加载失败时出现破图占位，以及重开文档后标题链接丢失标题视图状态的问题。
- 修复多块转换可能只修改部分选区，以及代码块转换为高亮块时丢失语言信息的问题。

## macOS 安装提示

当前发行构建使用免费 ad-hoc 签名，暂未使用 Apple Developer ID 签名和公证。将 AnnotaMD 拖入“应用程序”后，如 macOS 阻止启动，可执行：

```bash
xattr -cr /Applications/AnnotaMD.app
```

也可以在“系统设置 → 隐私与安全性”中选择“仍要打开”。

## 下载校验

所有安装包均记录在 Release 的 `SHA256SUMS.txt` 中，可在下载目录执行：

```bash
sha256sum -c SHA256SUMS.txt --ignore-missing
```
