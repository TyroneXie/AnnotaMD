# AnnotaMD 编辑控件统一设计

## 目标

在保留 MarkText 快编辑内核的前提下，统一代码块设置、代码块操作、源码模式入口和文字浮动工具栏，使默认阅读编辑路径更简洁，并保证用户设置与文字颜色可持久化。

## 已确认交互

1. “设置 → Markdown”新增“代码块”分组，包含“显示行号”和“自动换行”。“设置 → 编辑器 → 代码块”只保留代码字体、字号和原有空行处理。
2. 普通代码块右上角新增紧凑的“换行”按钮，与复制按钮并列。点击切换全局 `wrapCodeBlocks` 偏好；所有代码块同步更新，重启后保持。开启时按钮显示蓝色激活态。
3. 删除标题栏“块编辑 / 源码”分段按钮。每次启动都进入快编辑；源码模式继续由“视图 → 源码模式”和既有快捷键切换。
4. 浮动文字工具栏收紧到约 38px 高、单项约 30px。新增字体颜色和背景颜色面板，提供默认色与常用色板。
5. 字体颜色以 `<span style="color: …">…</span>` 保存，背景颜色以 `<span style="background-color: …">…</span>` 保存。重新打开、源码模式和 HTML/富文本导出均沿用现有原始 HTML 解析与清理链路。

## 数据与事件

- Muya 代码块按钮发出 `code-wrap-toggle`，payload 为 `{ enabled: boolean }`。
- 桌面编辑器收到事件后调用 `SET_SINGLE_PREFERENCE({ type: 'wrapCodeBlocks', value })`；现有 watcher 调用 `setOptions` 并更新根节点 `.mu-code-wrap`。
- 文字颜色继续走 `Format` 的 token/generator 管线，不另建文档外数据。

## 验证

- 设置项只在目标分组出现。
- 代码块按钮能切换、同步和重启保持。
- 标题栏不再出现模式分段按钮，菜单和快捷键仍能进入源码模式，重启回到快编辑。
- 颜色应用、替换、恢复默认、保存重开及导出均保留预期样式。
- 中英文文案和紧凑视觉在实际 Electron 窗口中检查。
