# Markdown Reader — 需求文档

## 1. 产品概述

Markdown Reader 是一款 macOS 原生应用，用于浏览和阅读 Markdown 文件。采用 SwiftUI + Textual 构建，提供两列布局：左侧目录树导航 + 右侧 Markdown 渲染显示，遵循 macOS 标准设计风格。

## 2. 目标用户

- 需要本地浏览 Markdown 文档的开发者和技术人员
- 需要阅读本地知识库/笔记的用户

## 3. 核心需求

### 3.1 目录树导航（左侧 Sidebar）

| 编号 | 需求 | 优先级 | 说明 |
|------|------|--------|------|
| ST-01 | 显示文件目录树 | P0 | 以树形结构展示当前打开目录下的 Markdown 文件和子目录；非 .md 文件在树中灰显显示 |
| ST-02 | 点击文件切换显示 | P0 | 点击目录树中的 Markdown 文件，右侧主体区切换显示对应内容 |
| ST-03 | 折叠/展开子目录 | P0 | 支持折叠和展开目录树中的子目录节点 |
| ST-04 | 显示/隐藏 Sidebar | P0 | 支持隐藏和显示左侧 Sidebar，可通过工具栏按钮或快捷键切换 |
| ST-05 | 目录图标区分 | P1 | 使用 SF Symbols 区分文件和目录图标 |
| ST-06 | 当前选中高亮 | P1 | 目录树中当前显示的文件应高亮标记 |
| ST-07 | 键盘导航 | P1 | 支持 ↑↓ 键在目录树中移动选中项，Enter 键打开/展开当前项 |

### 3.2 Markdown 显示区（右侧主体区）

| 编号 | 需求 | 优先级 | 说明 |
|------|------|--------|------|
| MD-01 | Markdown 渲染显示 | P0 | 默认以渲染后的富文本形式展示 Markdown 内容 |
| MD-02 | 原文/渲染切换 | P0 | 顶端提供「渲染」「原文」切换控件，支持在渲染视图和源码视图之间切换 |
| MD-03 | 默认渲染状态 | P0 | 打开文件后默认显示渲染状态 |
| MD-04 | 切换状态记忆 | P2 | 切换文件时记忆当前的渲染/原文状态 |
| MD-05 | 链接点击行为 | P1 | 渲染视图中的链接点击后在系统默认浏览器中打开 |
| MD-06 | 原文视图文本选择 | P1 | 原文视图支持文本选择和复制 |
| MD-07 | 原文视图 Word Wrap | P1 | 原文视图默认启用 Word Wrap，不出现横向滚动条 |

### 3.3 文件打开

| 编号 | 需求 | 优先级 | 说明 |
|------|------|--------|------|
| FO-01 | 打开目录 | P0 | 通过菜单或快捷键选择一个本地目录作为根目录 |
| FO-02 | 打开单个文件 | P1 | 支持直接打开单个 Markdown 文件，此时 Sidebar 不显示 |
| FO-03 | 拖拽打开 | P2 | 支持将目录或文件拖拽到应用窗口打开 |
| FO-04 | 最近打开记录 | P2 | 记录最近打开的目录/文件 |

### 3.4 搜索（后续迭代）

| 编号 | 需求 | 优先级 | 说明 |
|------|------|--------|------|
| SR-01 | 文件名搜索 | P2 | 在目录树中搜索文件名 |
| SR-02 | 全文搜索 | P3 | 在所有 Markdown 文件中搜索内容 |

### 3.5 空状态与异常处理

| 编号 | 需求 | 优先级 | 说明 |
|------|------|--------|------|
| EH-01 | 首次启动空状态 | P0 | 应用启动时无目录打开，右侧显示 Welcome 占位视图，提示用户打开目录 |
| EH-02 | 空目录状态 | P1 | 打开的目录中无 .md 文件时，右侧提示"该目录下无 Markdown 文件" |
| EH-03 | 文件读取失败 | P1 | 文件权限不足或编码异常时，显示错误提示而非崩溃 |
| EH-04 | 非 Markdown 文件点击 | P2 | 点击灰显的非 .md 文件，显示提示"不支持预览该文件类型" |
| EH-05 | 大文件性能 | P1 | 500KB 以上 Markdown 文件应保持滚动流畅（60fps 目标） |

## 4. 非功能需求

| 编号 | 需求 | 说明 |
|------|------|------|
| NF-01 | macOS 标准风格 | 遵循 Apple Human Interface Guidelines，使用原生控件和交互模式 |
| NF-02 | 原生性能 | 使用 SwiftUI 原生渲染，确保大文件滚动流畅 |
| NF-03 | 深色模式 | 支持 macOS 浅色/深色模式自动适配 |
| NF-04 | 最低系统版本 | macOS 15.0 (Sequoia)+ |

## 5. 约束

- 技术栈：SwiftUI + Textual（Swift 包，gonzalezreal/textual v0.3.1+）
- 仅支持 macOS 平台
- 只读浏览，不支持编辑
- 不做同步/云端功能
- Swift 6.0 + macOS 15.0 可使用 `@Observable` 宏（iOS 17+/macOS 14+）

## 6. 不做的范围（明确排除）

- Markdown 编辑功能
- 云端同步
- 导出为 PDF
- 多窗口（首版）
- 插件系统
