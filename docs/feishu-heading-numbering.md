# 飞书标题编号行为基线

本文记录 2026-07-20 在真实飞书文档中逐项编辑得到的结果，用于 AnnotaMD 标题编号交互的回归验证。结论来自实际输入，不是根据界面截图推测。

## 输入标准化规则

| 已有结构 | 新标题层级 | 用户输入 | 飞书结果 | 结论 |
| --- | --- | --- | --- | --- |
| `1.` | 一级 | `9.` | `2.` | 已有同级时，非 `1` 输入会续编 |
| `2.` | 二级首项 | `8.8.` | `2.8` | 父级前缀被校正，首个子项保留用户输入的末位 |
| `2.8` | 二级 | `3.3.` | `2.9` | 已有同级后忽略输入值，按前一同级末位加一 |
| `3.4` | 三级首项 | `6.6.6.` | `3.4.6` | 子级前缀始终跟随最近的已编号父级 |
| `3.4.6` | 三级 | `1.1.1.` | `3.4.1` | 输入末位为 `1` 表示明确重开当前层级 |
| `3.4.1` | 三级 | `9.9.9.` | `3.4.2` | 重开后从新序列继续递增 |
| `3.4.2` | 二级 | `9.9.` | `3.5` | 查找同级时跳过更深层标题 |
| `3.5`、未编号二级标题 | 二级 | `9.9.` | `3.6` | 未编号同级标题不中断编号 |
| `3.6`、未编号一级标题 | 二级 | `9.9.` | `3.7` | 未编号高层标题也不中断编号 |
| `3.` | 一级 | `1.` | `1.` | 输入末位 `1` 可重开顶层序列 |

中文句号 `。` 和全角句点 `．` 与英文句点 `.` 等价；例如 `1.1。` 加空格后应成为结构化编号并统一显示为 `1.1.`。

## 编号菜单规则

- 编号是独立、可点击的控件，不属于标题正文的普通文本选区。
- 第一个子标题如 `3.4` 没有可续接的同级前项，因此“继续标题编号”禁用。
- 同一位置仍可“重新开始编号”或“设置编号的值”；设置值只修改当前编号的最后一段。
- “继续标题编号”应只查找当前层级已有的编号，不应把父级的首个子编号当成可继续项。

## AnnotaMD 的实现边界

飞书将编号保存为块的结构化属性，可以在插入更高层编号后动态重排相关标题。AnnotaMD 必须保持 Markdown 纯文本兼容，因此编号最终仍写入标题文本。当前对齐范围是：输入空格时的标准化、同级续编、层级前缀、显式重开和编号菜单；不会在用户未编辑标题时批量改写整篇 Markdown。

## 回归检查

1. 用一级、二级、三级标题分别覆盖首个子项、同级续编和末位 `1` 重开。
2. 在两个编号标题之间插入未编号的同级与高层标题，确认序列仍连续。
3. 输入 `1.`、`1.1。`、`1。1。`，确认空格后统一显示英文点号且正文保留固定间距。
4. 点击蓝色编号，确认首个子项的“继续标题编号”禁用，“重新开始编号”和“设置编号的值”状态正确。
5. 运行 `npm run verify:menu`，并在 Electron 中用真实长文档复测。
